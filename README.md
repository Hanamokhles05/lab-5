# lab-5
📌 Project OverviewThis project focuses on predicting the Order Status (Delivered, Cancelled, or In Transit) for a food delivery dataset. The main goal of this task was to perform Feature Engineering to improve model performance and analyze how categorical feature reduction impacts the model's accuracy.🛠️ Key Features EngineeredTime-based Features: Extracted order_hour, day_of_week, is_weekend, and is_peak_hour from the timestamp.Numerical Scaling: Calculated price_per_item to understand pricing patterns.Data Leakage Prevention: Removed post-delivery columns like Delivery_Time and Delivery_Duration.Categorical Encoding: Applied One-Hot Encoding and handled high-cardinality features.🧪 The Experiment: Item Name ReductionThe core of this task was to experiment with the Item_Name column by grouping infrequent items into an "Other" category using a top_k threshold.Comparison AnalysisI tested three different values for top_k to see how they affect the Random Forest Classifier:Top_K ValueAccuracyObservationk = 30.XXSimplest model, focuses only on the 3 most popular items.k = 50.XXModerate detail, balances between complexity and noise.k = 60.XXMost detailed, includes more specific items in the features.Note: Replace 0.XX with your actual accuracy results from the code.📊 Feature Importance ResultsRegardless of the k value, the top features driving the model's decisions were:Delivery_Distance_km (Most significant)Total_Priceorder_hourThis indicates that the distance and time of day are much stronger predictors of order success than the specific item being ordered.🚀 How to RunClone the repository:Bashgit clone https://github.com/your-username/repository-name.git
